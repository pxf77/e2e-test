# 2.0.0 — Legacy Isolation

## Scope

Version 2.0 completes the physical separation between the generalized E2E framework and the historical insurance four-Agent runtime.

## Repository layout

```text
schemas/
  v1/                         # Legacy Agent and Skill contracts
  v2/                         # Current framework contracts

src/e2e_agent/
  workflow/                   # Current Workflow Runtime
  runners/                    # Current Runner adapters
  plugins/                    # Current Plugin SDK
  domains/                    # Current Domain Pack runtime
  legacy/
    agents/
    browser/
    graph/
    skills/
    cli.py
```

## Breaking changes

### Contract paths

Legacy contract files moved from:

```text
schemas/<contract>.schema.json
```

to:

```text
schemas/v1/<contract>.schema.json
```

Logical contract identities such as `merged-cases@v1` remain unchanged.

### Python imports

The following top-level Legacy packages were removed:

```text
e2e_agent.agents
e2e_agent.browser
e2e_agent.graph
e2e_agent.skills
e2e_agent.cli
e2e_agent.cli_entry
```

Use explicit imports:

```python
from e2e_agent.legacy.graph.graph import build_graph
from e2e_agent.legacy.graph.state import E2EAgentState
from e2e_agent.legacy.skills.loader import SkillPackageLoader
from e2e_agent.legacy.browser.runner import PlaywrightTSRunner
from e2e_agent.legacy import cli as legacy_cli
```

The installed command remains:

```text
e2e_agent.commands.main:main
```

### Tool commands

Root-level compatibility scripts were removed. Replace commands such as:

```bash
python tools/validate_schemas.py
python tools/acceptance_matrix.py
python tools/run_full_workflow.py --help
```

with:

```bash
python -m tools.validate.schemas
python -m tools.acceptance
python -m tools.legacy.run_full_workflow --help
```

Validation modules are available under `tools.validate`; diagnostics under `tools.diagnostics`.

### Gate CLI

The deprecated alias was removed:

```bash
e2e-agent gate-v2 ...
```

Use:

```bash
e2e-agent gate status <run-id>
e2e-agent gate approve <run-id>
e2e-agent gate reject <run-id>
e2e-agent gate resume <run-id>
```

The command detects Legacy and v2 checkpoints automatically.

## Retained compatibility

Version 2.0 still supports:

- `e2e-agent run --product-input <product-input.json>`
- insurance four-Agent execution
- Legacy R1–R4 checkpoint operations through unified `e2e-agent gate`
- Legacy Skill Packages beneath the explicit namespace
- Legacy Playwright compatibility execution beneath `e2e_agent.legacy.browser`

The compatibility surface is preserved, but its ownership is now explicit.
