# v1.1.0 — Repository Simplification

Release date: 2026-07-12

## Scope

1.1.0 simplifies the repository without removing the documented 1.x compatibility surface.

### Repository hygiene

- Generated runtime artifacts are rejected by CI.
- Obsolete patch history, duplicate run wrappers and stale implementation snapshots were removed.
- Example plugins and workflows are isolated under `examples/` and require explicit loading.

### Single sources of truth

- Insurance state dependencies and assertion templates live only in `domains/insurance/`.
- Skills are discovered from package-local `MANIFEST.yaml` files.
- Global `config/` is limited to framework model routing and Legacy Gate operations policy.
- `WorkflowRuntimeState` and `NodeRegistry` are the active runtime state and node registration models.

### Stable structure

- Canonical CLI: `e2e_agent.commands.main:main`.
- Documentation: `architecture/`, `guides/`, `reference/`, `sdk/`, `releases/`.
- Tools: `validate/`, `diagnostics/`, `legacy/`, `acceptance.py` with 1.x wrappers.
- Tests: `unit/`, `integration/`, `compatibility/`, `acceptance/`.

### Dependency and packaging audit

- Removed unused direct Python dependencies: `aiofiles`, `click`, `pydantic`.
- Removed duplicate direct Node dependency `playwright`; `@playwright/test` remains canonical.
- CI validates npm/uv locks, builds a wheel, installs it into an isolated environment and executes `e2e-agent --help`.

## Compatibility

The 1.x line continues to support:

- `e2e-agent run --product-input ...`
- insurance four-Agent Graph and Skill Packages
- v1 Gate checkpoints
- `cli_entry.py` and root tool compatibility wrappers
- `gate-v2` as a deprecated alias

New integrations should use App Pack, Domain Pack, Workflow DSL and the unified `e2e-agent gate` command.
